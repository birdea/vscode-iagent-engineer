# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |
| < 0.1.0 | No        |

## Reporting a Vulnerability

Report suspected vulnerabilities by opening a private security advisory in GitHub or by emailing the maintainer listed in the marketplace publisher profile.

Include:

- A short description of the issue
- Reproduction steps or a proof of concept
- Impact assessment
- Suggested remediation if available

You can expect an acknowledgement within 3 business days and a follow-up remediation plan after triage. Do not disclose the issue publicly until a fix or mitigation is available.
