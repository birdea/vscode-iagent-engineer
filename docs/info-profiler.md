# Profiler Screen Guide

## 한국어

### 개요

`iProfiler`는 저장된 AI agent 세션 로그를 VS Code 패널에서 빠르게 판독하기 위한 상세 분석 화면입니다. 상단 요약 카드, 중앙 차트, 하단 이벤트 로그를 한 화면에 배치해서 세션 규모, 토큰 흐름, 병목 구간, 원본 로그 위치를 함께 확인할 수 있습니다.

### 화면 구성

#### 1. Header

- 좌측 상단에는 agent vendor, 모델명, 세션 ID가 표시됩니다.
- `Info` 버튼은 현재 문서를 다시 여는 액션입니다.
- Live 세션을 보고 있는 경우 `Live` 배지가 표시되며, 클릭하면 실시간 모니터링을 중지합니다.

#### 2. Summary Metrics

- `File`: 현재 선택된 세션 파일명입니다.
- `Size`: 원본 세션 파일 크기입니다.
- `Tokens`: 입력 토큰과 출력 토큰의 합계를 나란히 보여줍니다.
- `Cost`: 모델명을 기반으로 추정한 대략적인 비용입니다.
- `Turns`: 요청/응답 턴 수 또는 타임라인 샘플 수입니다.
- `Duration`: 세션 시작부터 마지막 이벤트까지의 경과 시간입니다.
- `Peak`: 단일 포인트 기준 최대 토큰량입니다.
- `Avg/Turn`: 턴당 평균 토큰량입니다.
- `Cache`: 전체 토큰 대비 캐시 토큰 비율입니다.
- `Latency`: 가장 큰 응답 지연 시간입니다.
- `Total Tok`: 세션 전체 토큰 총합입니다.
- `Date`: 세션 시작 시각 기준 날짜입니다.

#### 3. Chart

- 기본 차트는 `Token Flow Comparison`이며, 세션 시계열에 따라 토큰 사용량을 보여줍니다.
- 범례 버튼 `Total`, `Output`, `Input`, `Cached`, `Trend`는 각각의 시리즈 표시 여부를 토글합니다.
- 막대는 각 타임라인 포인트의 대표 값을 나타내며, 원본 이벤트가 연결되어 있으면 클릭으로 소스 로그 라인으로 이동합니다.
- 툴팁은 현재 포인트의 시각과 시리즈 값을 보여주며, 연결된 원본이 있으면 바로 열 수 있습니다.

### 시리즈 정의

- `Total`: 입력, 출력, 캐시 토큰을 합친 총량입니다.
- `Output`: assistant 또는 agent가 생성한 출력 토큰입니다.
- `Input`: 사용자 입력 및 요청 컨텍스트에 사용된 입력 토큰입니다.
- `Cached`: 캐시 적중으로 재사용된 토큰입니다.
- `Trend`: 최근 포인트를 기준으로 계산한 이동 평균 추세선입니다.

### Event Log

- 시간순으로 정렬된 이벤트 목록입니다.
- `User`, `Agent`, `System` 역할로 구분해 어떤 종류의 이벤트인지 빠르게 파악할 수 있습니다.
- 메시지 미리보기, payload 크기, 토큰 수를 함께 표시합니다.
- 각 행을 클릭하면 해당 이벤트가 기록된 세션 파일의 원본 라인으로 이동합니다.

### 읽는 방법

1. 상단 Summary에서 세션 크기와 전체 비용 감을 먼저 확인합니다.
2. 차트에서 급격한 상승 구간이나 비정상적인 지연 구간을 찾습니다.
3. 툴팁이나 Event Log를 사용해 문제 시점의 원본 로그 라인으로 바로 이동합니다.
4. 캐시 비율과 평균 턴 토큰량을 같이 보면서 프롬프트 효율을 판단합니다.

### 참고

- 비용은 모델명 기준의 추정치이므로 실제 청구액과 정확히 일치하지 않을 수 있습니다.
- 세션 파일 포맷에 따라 일부 값은 추론되거나 비어 있을 수 있습니다.
- Live 모드에서는 파일이 갱신될 때마다 차트와 이벤트 로그가 다시 계산됩니다.

## English

### Overview

`iProfiler` is the detailed analysis surface for archived AI agent sessions inside the VS Code panel area. It combines a compact summary, a time-series chart, and a raw event log so you can read session scale, token usage, latency spikes, and source-log references in one place.

### Layout

#### 1. Header

- The top-left block shows the agent vendor, model name, and session ID.
- The `Info` button reopens this guide.
- When live monitoring is active, a `Live` badge appears and can be clicked to stop the live stream.

#### 2. Summary Metrics

- `File`: the selected session file name.
- `Size`: the size of the source session file.
- `Tokens`: input and output token totals shown side by side.
- `Cost`: an estimated cost derived from the detected model name.
- `Turns`: the number of requests/responses or timeline samples.
- `Duration`: elapsed time from the first event to the last one.
- `Peak`: the highest token volume seen in a single point.
- `Avg/Turn`: the average token volume per turn.
- `Cache`: cached-token ratio across the whole session.
- `Latency`: the largest observed response delay.
- `Total Tok`: total tokens for the session.
- `Date`: the session start date.

#### 3. Chart

- The default chart is `Token Flow Comparison`, which plots token usage across the session timeline.
- Legend buttons `Total`, `Output`, `Input`, `Cached`, and `Trend` toggle each series on or off.
- Bars represent the primary value for each timeline point and can jump back to the linked source log line when available.
- The tooltip shows the point timestamp and all visible series values, and can open the linked source record directly.

### Series Definitions

- `Total`: the combined input, output, and cached token volume.
- `Output`: tokens generated by the assistant or agent.
- `Input`: tokens consumed by user input and request context.
- `Cached`: tokens reused through cache hits.
- `Trend`: a moving-average trend line based on recent points.

### Event Log

- The event log is ordered by time.
- Roles are grouped as `User`, `Agent`, and `System` for faster scanning.
- Each row includes a short preview, payload size, and token count when available.
- Clicking a row opens the original session file at the matching source line.

### Suggested Reading Flow

1. Start with the summary metrics to gauge session size and cost.
2. Scan the chart for spikes, drops, and suspicious latency stretches.
3. Use the tooltip or event log to jump straight to the raw source lines around those moments.
4. Compare cache ratio and average tokens per turn to evaluate prompt efficiency.

### Notes

- Cost is an estimate based on the detected model name and may not match billing exactly.
- Some values can be inferred or unavailable depending on the session file format.
- In live mode, the chart and event log are recomputed whenever the source file changes.
